Little productivity boosters - git alias

Seb Dalgarno / 2021-01-12

The problem

For years I have staged, committed and pushed changes with the RStudio Git interface.

It works! But it can be time-consuming when you are doing it a lot. And I’ve been trying to use my mouse less. It would be better to do this in the terminal.

The solution

I found this solution on stackoverflow, which suggests staging and committing all files by running the following in the terminal:

git add -A && git commit -m "rebuild site"

We can push like this:

git push

However, the real productivity gains are made by using git aliases, which save us having to type (and remember) full git commands. Let’s start with a simple example. To set up the alias git p for git push, we type in the terminal:

git config --global alias.p "push"

To set up the alias git c for git commit we do:

git config --global alias.c "commit"

Now going back to our previous case of stage/commit all, we can set up the alias git coa:

git config --global alias.coa "!git add -A && git commit -m"

From now on, we can stage, commit and push all changes with two simple commands:

git coa 'rebuild site'

git p

Now to completely remove the need to use the mouse, we can set up a customized keyboard shortcut1 Cmd-3 to move the cursor to the RStudio terminal.2 The advantage of using the RStudio terminal and not the Terminal app (or iterm2)3 is that it is automatically drilled down into the directory of your project.
